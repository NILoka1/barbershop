# 🌐 Переменные окружения

Этот проект использует `.env`-файлы для управления конфигурацией в разных окружениях.

## 📄 Основные переменные

Переменные определены в `.env.example` и используются в `server` и `client`.

### `server/.env` (или корневой `.env`)

| Переменная | Описание | Пример |
|------------|---------|--------|
| `PORT` | Порт сервера | `3001` |
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql://postgres:postgres@localhost:5432/barbershop` |
| `JWT_SECRET` | Секрет для подписи токенов | `your-super-secret-jwt-key` |
| `CLIENT_URL` | Разрешённый origin для CORS | `http://localhost:5173` |
| `CLIENT_BUILD_URL` | URL продакшен-фронтенда | `https://barbershop.example.com` |
| `ZEROTIRE_URL` | URL для preview-деплоя | `https://barbershop-xyz.z01.de` |
| `NODE_ENV` | Окружение | `development` или `production` |

> ⚠️ Не храните `JWT_SECRET` и `DATABASE_URL` в репозитории!

## 🔐 Валидация переменных

Рекомендуется добавить валидацию при старте сервера:

```ts
// packages/server/src/env.ts
import { z } from 'zod';

const envSchema = z.object({
  PORT: z.string().default('3001'),
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
  CLIENT_URL: z.string().url().optional(),
});

export const env = envSchema.parse(process.env);
```

И импортировать в `index.ts`:
```ts
import { env } from './env';
```

## 🐳 Docker

В `docker-compose.yml` переменные передаются в контейнеры:
- `postgres` — через `environment`
- `db-seed` — через `DATABASE_URL`

> 💡 Совет: Используйте `dotenv` в обоих сервисах, но не коммитьте `.env`.
