## 🔄 Вечный ререндер модалок

### Симптомы
- Модалки перерисовываются бесконечно.
- Высокая нагрузка на CPU.

### Возможные причины
- `React.StrictMode` в `main.tsx` вызывает двойной рендер.
- `zustand`-стор обновляется в рендере.
- `Mantine ModalsProvider` пересоздаётся.

### Решение
1. Убедитесь, что `ModalsProvider` не пересоздаётся:
```tsx
<ModalsProvider>  <!-- Должен быть статичным -->
  <App />
</ModalsProvider>
```
2. Проверьте, нет ли вызова `openModal` в рендере.
3. Временно отключите `StrictMode` для теста.

## ✨ Мерцания при удалении

### Симптомы
- Элемент моргает при удалении.
- После удаления кратковременно появляется снова.

### Причина
- Данные удаляются, но `React Query` не инвалидирует кеш.

### Решение
После мутации — инвалидируйте запрос:
```ts
const { mutate } = trpc.worker.delete.useMutation({
  onSuccess: () => {
    queryClient.invalidateQueries(['workers']); // или используйте queryKey
  },
});
```

## 🔐 Проблемы с CORS

### Симптомы
- Ошибка `CORS` при запросе с фронтенда.

### Причина
- В `allowedOrigins` есть пустая строка `''`.

### Решение
Удалить `''` из массива в `server/src/index.ts`.

## 📁 Циклические зависимости

### Риск
- `shared` импортирует из `server` → цикл.

### Решение
- Удалить `server: workspace:*` из `devDependencies` в `shared`.
- Никогда не импортировать реализацию в `shared`.
