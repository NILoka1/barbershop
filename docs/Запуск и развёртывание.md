# 🚀 Запуск и развёртывание

Как запустить проект локально и в продакшене.

## 🖥️ Локальный запуск

### Предварительные требования
- `Node.js` (v18+)
- `pnpm`
- `Docker` и `Docker Compose`

### Шаги

1. Установите зависимости:
```bash
pnpm install
```

2. Запустите базу данных:
```bash
docker-compose up -d postgres
```

3. Примените миграции:
```bash
pnpm --filter server prisma:migrate
```

4. Запустите сиды:
```bash
pnpm --filter server seed
```

5. Запустите оба сервиса:
```bash
pnpm dev
```
> Эквивалентно: `pnpm --filter server dev & pnpm --filter client dev`

Фронтенд: `http://localhost:5173`
Сервер: `http://localhost:3001`

## 🌐 Продакшен

### Сборка

```bash
# Сборка клиента
pnpm --filter client build

# Сервер можно запускать из исходников (ts-node) или собрать
```

### Запуск

```bash
# Запустить сервер
pnpm --filter server start

# Раздать статику клиента через Nginx или Express
```

## 🐳 Docker (опционально)

Можно создать отдельный `Dockerfile` для фронтенда и сервера, но текущая схема:
- БД — в Docker
- Приложение — локально

Для полного деплоя:
- Создайте `Dockerfile` для `server` и `client`
- Соберите образы
- Запустите через `docker-compose.prod.yml`
