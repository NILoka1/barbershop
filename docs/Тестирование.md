# 🧪 Тестирование

Документация по тестированию в проекте.

## 📌 Текущее состояние

- **Есть unit-тесты** в клиентском приложении, использующие Vitest
- **Нет тестов** на стороне сервера и в shared-пакете
- Используется Vitest (не Jest) для тестирования
- Есть базовая конфигурация в `vitest.config.ts`
- Используются Testing Library для компонентных тестов

## 🎯 Цель

Расширить и унифицировать тестирование по всему проекту:
- Добавить unit-тесты для shared-пакета
- Настроить тестирование на сервере
- Добавить интеграционные тесты для API
- Добавить e2e тесты для критичных пользовательских сценариев

## 🛠️ Используемые инструменты

| Тип тестов       | Инструмент      | Назначение |
|------------------|----------------|-----------|
| Unit (client)    | Vitest         | Тестирование компонентов, хуков |
| Unit (shared)    | Vitest         | Тестирование утилит, валидации |
| Unit (server)    | Vitest         | Тестирование сервисов, утилит |
| Компонентные     | @testing-library/react | Тестирование React-компонентов |
| Интеграционные (API) | Supertest + Vitest | Проверка tRPC-роутов, авторизации |
| E2E (UI)         | Playwright     | Проверка пользовательских сценариев |
| Линтинг          | eslint --fix   | Качество кода |

## 🔐 Тестирование tRPC

Пример теста для `auth.login` на сервере:

```ts
// packages/server/src/test/integration/auth.test.ts
import request from 'supertest';
import { app } from '../../src/index';
import { createContext } from '../../src/trpc';

describe('Auth Router', () => {
  it('должен входить и возвращать токен', async () => {
    const response = await request(app)
      .post('/trpc/auth.login')
      .send({ email: 'admin@barber.com', password: '123' });
    
    expect(response.status).toBe(200);
    expect(response.body.result.data.token).toBeDefined();
  });
});
```

## 🧩 Тестирование компонентов

Пример теста компонента в клиенте:

```tsx
// packages/client/src/components/Button.test.tsx
import { describe, it, expect } from "vitest";
import { screen } from "@testing-library/react";
import { renderWithProviders } from "../test/test-utils";
import Button from "./Button";

describe("Button", () => {
  it("рендерит children", () => {
    renderWithProviders(<Button>Текст кнопки</Button>);
    expect(screen.getByText("Текст кнопки")).toBeInTheDocument();
  });
});
```

## 📊 Покрытие кода

Настроить `vitest --coverage` для отслеживания % покрытия. Запуск:
```
npm run test:coverage
```
