# 🧪 Тестирование

План по внедрению тестирования в проект.

## 📌 Текущее состояние

- **Нет тестов** — ни unit, ни интеграционных.
- Нет тестовых фреймворков в `devDependencies`.
- Нет папки `tests` или `__tests__`.

## 🎯 Цель

Внедрить **полный набор тестов**: unit, интеграционные, e2e.

## 🛠️ Рекомендуемые инструменты

| Тип тестов       | Инструмент      | Назначение |
|------------------|----------------|-----------|
| Unit (shared)    | Vitest         | Тестирование типов, утилит |
| Интеграционные (API) | Supertest + Jest | Проверка tRPC-роутов, авторизации |
| E2E (UI)         | Playwright     | Проверка пользовательских сценариев |
| Линтинг          | eslint --fix   | Качество кода |

## 🔐 Тестирование tRPC

Пример теста для `auth.login`:

```ts
// packages/server/tests/auth.test.ts
import request from 'supertest';
import { app } from '../src/index';

describe('Auth Router', () => {
  it('должен входить и возвращать токен', async () => {
    const response = await request(app)
      .post('/trpc/auth.login')
      .send({ email: 'admin@barber.com', password: '123' });
    expect(response.body.result.data.token).toBeDefined();
  });
});
```
